<inner> quintroCellsFromRange(args)

Convenience function for transforming a string into a range of cell coordinates.

Parameters:
Name Type Description
args object

the function arguments

Properties
Name Type Argument Default Description
range string

a string representing a range to transform into cell coordinates Range is a string of the form "[column range],[row range]", where [column range] and [row range] are each strings that match the pattern /^\d+(-\d+)?$/. If only one number is specified, that part of the coordinate stays constant for all elements in the return value. Otherwise, the return value includes a cell coordinate for each number in the range from the first number to the last number, inclusive, increased by 1 each time. If the first number is greater than the second number, then that part of the cell coordinates (column or row) will decrease by 1 each time, instead of increasing by 1.

Examples:

"1-6,6-1" => [
     { position: [1, 6] }, 
     { position: [2, 5] }, 
     { position: [3, 4] }, 
     { position: [4, 3] }, 
     { position: [5, 2] }, 
     { position: [6, 1] }
]

"3-7,5"   => [
     { position: [3, 5] }, 
     { position: [4, 5] }, 
     { position: [5, 5] }, 
     { position: [6, 5] }, 
     { position: [7, 5] }
]

"6,4-9"   => [
     { position: [6, 4] }, 
     { position: [6, 5] }, 
     { position: [6, 6] }, 
     { position: [6, 7] }, 
     { position: [6, 8] }, 
     { position: [6, 9] }
]

If a cell is represented in the filledMap, its color is added to the cell coordinate under the key "color" (e.g. { position: [1, 2], color: "blue" })
board shared-lib.Board <optional>

the board in which the cells exist
positionsOnly boolean <optional>
 false

if true, will return just the positions, rather than objects with a position key (a la Type.Cell)
Returns:

list of cells from the specified range, either as an array of Types.Cells containing only a position key, or an array of two-element tuples corresponding to the positions

Type
Array.<Types.Cell> | Array.<Array.<number>>
